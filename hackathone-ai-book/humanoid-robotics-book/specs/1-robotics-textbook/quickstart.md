# Quickstart Guide: Physical AI & Humanoid Robotics Textbook\n\n**Feature Branch**: `1-robotics-textbook` | **Date**: 2025-12-05 | **Plan**: [specs/1-robotics-textbook/plan.md](specs/1-robotics-textbook/plan.md)\n\n## Overview\n\nThis guide provides instructions for setting up your development environment to contribute to the \"Physical AI & Humanoid Robotics: A Comprehensive Textbook\". It covers the necessary tools for both content development (Docusaurus) and robotics-specific components (ROS 2, simulation environments).\n\n## 1. Prerequisites\n\nEnsure you have the following installed:\n\n-   **Git**: For version control.
-   **Node.js & npm**: Required for Docusaurus.
-   **Python 3.10+**: For all robotics code examples.
-   **ROS 2 (Humble or Iron)**: For running ROS 2 specific examples. Follow official installation guides for your Linux distribution.
-   **Docker (Recommended)**: For creating reproducible development environments for ROS 2 and simulations.
-   **Gazebo or Isaac Sim**: For running simulation exercises. Refer to their official documentation for installation.\n\n## 2. Setting Up the Docusaurus Environment\n\n1.  **Clone the Repository**:\n    ```bash\n    git clone [repository-url]\n    cd humanoid-robotics-book\n    ```\n2.  **Install Docusaurus Dependencies**:\n    ```bash\n    npm install\n    ```\n3.  **Start Local Development Server**:\n    ```bash\n    npm run start\n    ```\n    This will open the textbook in your browser at `http://localhost:3000`. Changes to Markdown files will live-reload.\n\n## 3. Contributing to Content\n\n-   **Writing Modules**: New textbook modules should be created as `.mdx` files in the `docs/` directory, following the existing structure (e.g., `docs/module-XX/index.mdx`).\n-   **Adding Diagrams**: Place SVG or PNG diagrams in `static/images/` and reference them in your Markdown files.\n-   **Code Examples**: Python code for ROS 2 examples should reside in `ros2_packages/`. Simulation assets (e.g., world files, robot models) should be placed in `simulations/`.\n-   **Glossary Terms**: Add new terms and definitions to the glossary Markdown file in `glossary/terms.md`.\n\n## 4. Building for Deployment\n\nTo build the static Docusaurus site for deployment (e.g., to GitHub Pages):\n\n```bash\nnpm run build\n```\n\nThe generated static files will be in the `build/` directory.\n
